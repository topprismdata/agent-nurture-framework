# Acknowledgments

The Agent Nurture Framework builds on a rich body of prior work across knowledge management, cognitive science, and AI agent architecture. We are grateful to the researchers and practitioners whose contributions made this framework possible.

## Foundational Theory

**Zhang, K. (2026)** -- *Nurture-First Development: A Knowledge Crystallization Approach to Domain-Specific AI Agent Development* (arXiv:2603.10808). This paper provides the foundational theoretical framework for the Agent Nurture Framework, including the Knowledge Crystallization Cycle, the three-layer cognitive architecture, and the formal model that underpins all operational patterns in this project.

**Nonaka, I. & Takeuchi, H. (1995)** -- *The Knowledge-Creating Company*. The SECI model of knowledge creation (Socialization, Externalization, Combination, Internalization) directly informs the design of the crystallization cycle, particularly the transformation from tacit to explicit knowledge that lies at the heart of the framework.

**Dreyfus, H.L. & Dreyfus, S.E. (1986)** -- *Mind Over Machine*. The Dreyfus five-stage skill acquisition model provides the theoretical basis for the capability rating system and the understanding that expertise development involves qualitative restructuring of knowledge, not merely accumulation of rules.

**Collins, A., Brown, J.S. & Newman, S.E. (1989)** -- *Cognitive Apprenticeship*. The cognitive apprenticeship model informs the framework's approach to knowledge transfer through modeling, coaching, scaffolding, and fading, which map directly onto the nurture-first developmental phases.

**Kolb, D.A. (1984)** -- *Experiential Learning: Experience as the Source of Learning and Development*. Kolb's four-stage learning cycle provides the theoretical grounding for the Five-Stage Learning Loop, establishing the cyclical relationship between experience, reflection, conceptualization, and experimentation.

**Polanyi, M. (1966)** -- *The Tacit Dimension*. Polanyi's distinction between tacit and explicit knowledge motivates the entire nurture-first approach: the recognition that the most valuable domain knowledge often cannot be written down, only transferred through sustained practice and immersion.

## Frameworks and Architectures

**tanaikech (2026)** -- *Recursive Knowledge Crystallization (RKC) Framework*. The RKC framework introduced the concept of persistent, version-controlled knowledge crystallization through recursive interaction cycles. The Agent Nurture Framework generalizes this approach from a single-file model to a three-layer architecture while preserving the core insight of compounding knowledge returns.

**Bhargava, S. (2026)** -- *Modular Intelligence: A Skill-Based Paradigm for Scalable AI Agent Architecture*. The skill-based paradigm and Single Responsibility Principle for agent skills directly informed the design of the skill layer, particularly the emphasis on composability, explicit interfaces, and independent evolution of knowledge modules.

## Agent Architectures

**The MemGPT team** (Packer et al., 2023) -- MemGPT's hierarchical memory management system demonstrated the value of differentiated access patterns for agent memory, inspiring aspects of the three-layer architecture's loading and retrieval strategies.

**The Voyager team** (Wang et al., 2023) -- Voyager's skill library for lifelong learning agents provided the closest architectural precedent to the skill layer, demonstrating that reusable skill libraries enable compounding capability growth.

**The Generative Agents team** (Park et al., 2023) -- The reflection mechanism in generative agents, which synthesizes higher-level observations from accumulated experience, is closely related to the crystallization operation and informed its design.

## Community

We also acknowledge the broader **AI agent development community** -- researchers, practitioners, and open-source contributors who are advancing the state of the art in agent architecture, memory systems, and knowledge management. This framework is part of a collective effort to move beyond static agent configurations toward agents that genuinely learn and grow through sustained collaboration with their users.
