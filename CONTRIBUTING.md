# Contributing to Agent Nurture Framework

Thank you for your interest in contributing!

## How to Contribute

1. **Fork** the repository
2. **Create a feature branch** (`git checkout -b feature/your-feature`)
3. **Commit your changes** with clear, descriptive messages
4. **Push** to your fork
5. **Open a Pull Request** with a description of your changes

## Areas We Welcome Contributions

- **Bug fixes** for scripts or documentation errors
- **New templates** for additional use cases or platforms
- **Case studies** from other domains applying the framework
- **Translations** of documentation into other languages
- **Script improvements** with better error handling or new features
- **Platform-specific guides** for Codex CLI, Gemini CLI, or other agent platforms

## Documentation Standards

- Use consistent formatting (YAML frontmatter + structured Markdown)
- Include cross-references as clickable Markdown links
- Keep domain-specific examples in appropriate sections (case studies, examples)
- Maintain the generalization principle: core docs should be domain-agnostic

## Reporting Issues

Please use GitHub Issues with:
- Clear description of the problem or suggestion
- Steps to reproduce (for bugs)
- Expected vs. actual behavior
